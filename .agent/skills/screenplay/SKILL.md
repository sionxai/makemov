# Screenplay Writing

## 목표
확정된 시놉시스를 촬영/편집 가능한 씬 단위 시나리오로 변환한다.

## 입력 계약
필수 입력:
- 승인된 시놉시스 JSON (`core_scenes`, `characters`, `visual_tone` 포함)

선택 입력:
- 대사 밀도(낮음/중간/높음)
- 연출 디테일 수준(간결/표준/상세)
- 러닝타임 우선순위(정확한 길이 vs 감정 완성도)

## 출력 계약
항상 아래 2가지를 함께 출력한다.

1) 읽기용 시나리오
- 씬 헤더: `S#[번호]. INT/EXT. 장소 - 시간`
- 타임코드: `[TC 00:00.0~00:18.5]`
- 액션/대사/SFX/전환

2) 기계 처리용 JSON
```json
{
  "title": "",
  "target_runtime_sec": 0,
  "scenes": [
    {
      "scene_id": "S1",
      "slugline": "EXT. LOCATION - DAWN",
      "tc_start": "00:00.0",
      "tc_end": "00:18.5",
      "duration_sec": 18.5,
      "objective": "",
      "conflict": "",
      "turn": "",
      "beats": [
        {
          "beat_id": "S1-B1",
          "type": "action",
          "duration_sec": 4.0,
          "content": "",
          "camera_hint": ""
        }
      ],
      "transition_out": "CUT TO"
    }
  ],
  "assumptions": []
}
```

## 타임코드 산정 규칙
- 대사 비트 길이: `max(2.0, 글자수 / 6.0)`
- 액션 비트 길이: `max(2.0, 문장수 * 2.5)`
- 씬 길이: 비트 길이 합계(소수점 1자리)
- 전체 길이가 목표와 10% 이상 차이나면 비트 길이를 비례 보정한다.

## 작성 워크플로
1. 시놉시스의 `core_scenes`를 씬 골격으로 고정한다.
2. 각 씬에 `objective`, `conflict`, `turn`을 먼저 작성한다.
3. 비트(action/dialogue/sfx/transition)로 세분화한다.
4. 타임코드를 계산하고 씬 간 연속성을 점검한다.
5. 읽기용 시나리오와 JSON을 동기화한다.
6. 체크리스트 검수 후 제출한다.

## 결정 규칙
- 장면 번호는 `S1`부터 순차 유지한다.
- 한 비트에는 핵심 행동 1개만 배치한다.
- 내면 상태는 표정/행동/대사로 변환해서 쓴다.
- 카메라 지시는 필수 아님이다. 필요할 때만 `camera_hint`에 기록한다.

## 품질 게이트
- 모든 씬이 이야기 진행에 필요한가?
- 씬마다 목표와 전환점이 명확한가?
- 타임코드 합계가 목표 러닝타임에 수렴하는가?
- 대사가 설명문이 아니라 갈등/관계를 드러내는가?
- 다음 단계(스토리보드)가 JSON만으로 컷 분해 가능한가?

## 금지
- 촬영 불가능한 추상 내면 문장
- 목적 없는 분위기용 씬
- 타임코드 없는 대사/액션 나열

## 다음 단계 Handoff
- `scenes[]`를 스토리보드 스킬 입력으로 전달한다.
- 변경 요청 시 JSON과 읽기용 본문을 함께 업데이트한다.
